# 알아야할 개념

* DB-index: 트리
  - 이진트리: 중간에 있는 포인트를 이용해, 검색 범위를 반으로 줄이는 작업
    - 순차적으로 진행할 경우: 평균 50만번
    - 이진트리의 경우: 100만개 -> 50만개 -> 25만개 ... : 로그 n번을 n번으로 검색 -> 20번
    => 검색시간을 획기적으로 줄일 수 있음!
